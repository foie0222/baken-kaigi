## US-02-001: 負け額限度額の初期設定

**ストーリー**
> 新規ユーザーとして、アカウント登録時に月間の負け額限度額を設定したい。なぜなら、最初から自分の許容範囲を明確にして責任ある賭けを始めたいから。

| 項目 | 内容 |
|------|------|
| 優先度 | Must have |
| 認証 | **必要** |
| 対象ペルソナ | 全ペルソナ（特にペルソナ1） |

**受け入れ条件**
- [ ] アカウント登録フロー内で負け額限度額の設定画面が表示される
- [ ] 限度額は必須入力（スキップ不可）
- [ ] 推奨範囲（例: 月収の5%以内）が参考として表示される
- [ ] 金額はプリセット（¥10,000, ¥30,000, ¥50,000, ¥100,000）または直接入力できる
- [ ] 設定完了後、確認画面が表示される
- [ ] 設定した限度額はいつでも設定画面から確認できる

---

## US-02-002: 負け額限度額の変更

**ストーリー**
> ログインユーザーとして、負け額限度額を変更したい。なぜなら、自分の経済状況に合わせて調整したいから。

| 項目 | 内容 |
|------|------|
| 優先度 | Should have |
| 認証 | **必要** |
| 対象ペルソナ | 全ペルソナ |

**受け入れ条件**
- [ ] 設定画面から負け額限度額を変更できる
- [ ] 減額は即時反映される
- [ ] 増額には7日間の待機期間が必要
- [ ] 増額申請時に「衝動的な変更ではありませんか？」の確認メッセージが表示される
- [ ] 待機期間中は申請中であることが表示される
- [ ] 待機期間経過後に自動的に新しい限度額が適用される

---

## US-02-003: 負け額限度額到達時の強制停止

**ストーリー**
> 負け額限度額に到達したユーザーとして、その月は賭けができないようにしてほしい。なぜなら、自分で設定した限度額を絶対に守りたいから。

| 項目 | 内容 |
|------|------|
| 優先度 | **Must have** |
| 認証 | **必要** |
| 対象ペルソナ | 全ペルソナ（特にペルソナ1） |

**受け入れ条件**
- [ ] 負け額限度額に到達すると購入機能が即座に無効化される
- [ ] 購入ボタンは非活性化され、クリックできない
- [ ] 「今月の負け額限度額に到達しました」のメッセージが表示される
- [ ] 翌月1日0:00に自動的に制限が解除される
- [ ] 制限中もレース情報の閲覧、AI対話は利用可能
- [ ] 回避方法は一切提供しない（システムレベルで強制）

---

## US-02-004: 負け額限度額接近時のアラート表示

**ストーリー**
> ログインユーザーとして、負け額限度額に近づいたときに警告を受けたい。なぜなら、限度額に到達する前に自制したいから。

| 項目 | 内容 |
|------|------|
| 優先度 | Must have |
| 認証 | **必要** |
| 対象ペルソナ | 全ペルソナ（特にペルソナ1） |

**受け入れ条件**
- [ ] 負け額が限度額の80%に到達するとアラートが表示される
- [ ] アラートは画面上部に大きく表示され、見逃せない設計
- [ ] 残り許容負け額が金額で表示される
- [ ] 現在の購入で限度額に到達する場合は赤色で警告表示される
- [ ] アラートは非表示にできない（常に表示）
- [ ] アラートをタップすると損益ダッシュボードに遷移できる

---

## US-02-005: 損益ダッシュボードの表示

**ストーリー**
> ログインユーザーとして、自分の損益をダッシュボードで確認したい。なぜなら、賭けの全体像を把握して自己管理したいから。

| 項目 | 内容 |
|------|------|
| 優先度 | Should have |
| 認証 | **必要** |
| 対象ペルソナ | 全ペルソナ |

**受け入れ条件**
- [ ] ダッシュボード画面で今月の損益サマリーが表示される
- [ ] 当月の負け額限度額と使用状況がプログレスバーで表示される
- [ ] 残り許容負け額が金額で表示される
- [ ] 日次/週次/月次/累計の表示切り替えができる
- [ ] 各期間の総投資額、総払戻額、純損益が表示される
- [ ] 過去データは期間無制限で閲覧可能

---

## US-02-006: 損益グラフの表示

**ストーリー**
> ログインユーザーとして、損益の推移をグラフで視覚的に確認したい。なぜなら、自分の賭けパターンの傾向を把握したいから。

| 項目 | 内容 |
|------|------|
| 優先度 | Could have |
| 認証 | **必要** |
| 対象ペルソナ | 全ペルソナ（特にペルソナ2） |

**受け入れ条件**
- [ ] 損益推移が折れ線グラフで表示される
- [ ] 日次/週次/月次/累計で表示期間を切り替えられる
- [ ] グラフの各ポイントをタップすると詳細が表示される
- [ ] 勝率（当選/購入回数）が表示される
- [ ] 平均掛け金額が表示される
- [ ] 回収率（払戻金/投資額）が表示される

---

## US-02-007: 賭け履歴一覧の表示

**ストーリー**
> ログインユーザーとして、過去の全賭け履歴を確認したい。なぜなら、自分の賭け傾向を分析して改善したいから。

| 項目 | 内容 |
|------|------|
| 優先度 | Should have |
| 認証 | **必要** |
| 対象ペルソナ | 全ペルソナ |

**受け入れ条件**
- [ ] 賭け履歴が新しい順に一覧表示される
- [ ] 各履歴にレース名、日付、買い目、金額、結果が表示される
- [ ] フィルタ機能で日付範囲を絞り込める
- [ ] フィルタ機能でレース場を絞り込める
- [ ] フィルタ機能で券種（単勝、馬連など）を絞り込める
- [ ] 履歴の保存期間は無制限
- [ ] 履歴をタップすると詳細（オッズ、払戻金など）が表示される

## US-02-008: 依存症相談窓口情報の表示

**ストーリー**
> ユーザーとして、ギャンブル依存症の相談窓口情報にアクセスしたい。なぜなら、必要なときに専門家の支援を受けられるようにしたいから。

| 項目 | 内容 |
|------|------|
| 優先度 | Should have |
| 認証 | 不要 |
| 対象ペルソナ | 全ペルソナ |

**受け入れ条件**
- [ ] フッターに「ギャンブル依存症相談」リンクが常に表示される
- [ ] 設定画面に「依存症相談窓口」セクションがある
- [ ] 公的機関（精神保健福祉センター等）の連絡先が掲載される
- [ ] 電話番号はタップで発信できる
- [ ] ウェブサイトはタップで外部ブラウザで開ける
- [ ] 認証不要でアクセス可能

## US-03-005: 購入時の負け額限度額チェック

**ストーリー**
> ログインユーザーとして、購入時に負け額限度額を超えないかチェックしてほしい。なぜなら、限度額を超える購入を未然に防ぎたいから。

| 項目 | 内容 |
|------|------|
| 優先度 | **Must have** |
| 認証 | **必要** |
| 対象ペルソナ | 全ペルソナ（特にペルソナ1） |
| 備考 | EPIC-03から本ユニットに移動 |

**受け入れ条件**
- [ ] 購入確認時に残り許容負け額との比較が表示される
- [ ] 購入金額が残り許容負け額を超える場合は警告が表示される
- [ ] 超過時は「金額を調整してください」のメッセージが表示される
- [ ] 超過時は「購入確定」ボタンが無効化される
- [ ] 負け額限度額到達済みの場合は購入画面に遷移できない
- [ ] 到達済みの場合は「今月は購入できません」のメッセージが表示される
